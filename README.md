# DL-Feature-Pipeline  
**面向深度学习的自动化特征工程与训练流水线**

> 一个以 **特征工程 Agent** 为核心的自动化数据处理工作流，结合深度学习训练与评估系统，通过归一化、标准化和对数变换等策略，构建「数据感知—特征变换—模型训练—效果反馈」的完整闭环。

---

## 项目简介

本项目基于 **California Housing 数据集**，构建了一套自动化的特征工程与深度学习训练流水线，重点围绕 **归一化（Min-Max）、标准化（Z-score）与对数变换** 的协同使用展开。

通过多 Agent 协作与定时实验调度，系统能够在不同数据集上自动选择并验证最合适的特征变换策略，持续优化模型收敛速度与预测精度，形成「统计分析 → 特征变换 → 模型训练 → 评估反馈 → 策略迭代」的自驱闭环。

---

## 核心架构

工作流拆分为多 Agent 协同：

- **Profiler Agent**：  
  负责数据统计与分布识别，自动检测特征是否存在明显边界、是否呈右偏分布、是否近似正态分布，为特征变换提供依据。

- **Transformer Agent**：  
  根据 Profiler 的分析结果，自动选择并执行合适的变换策略：
  - 对有明显边界的特征（如经纬度）使用 **Min-Max 归一化**
  - 对近似正态分布的特征（如收入）使用 **Z-score 标准化**
  - 对强右偏的目标变量（如房价）使用 **对数变换**

- **Evaluator Agent**：  
  负责训练监控与效果评估，跟踪训练损失、RMSE、收敛速度等指标，并根据实验结果反向调整特征工程策略与超参数组合。

---

## 核心功能

- **自适应特征工程**  
  根据数据分布自动匹配合适的归一化 / 标准化 / 对数变换策略，减少人工干预。

- **深度学习训练优化**  
  通过合理的特征变换提升数值稳定性，加快模型收敛，降低预测误差。

- **自动化实验调度**  
  支持通过配置文件与定时任务（cron / 任务计划程序）持续运行，单日可完成 **10+ 组不同特征工程方案的对比实验**。

- **闭环自迭代**  
  持续采集训练指标与评估结果，反向优化特征处理策略，形成可复用的经验积累。

## 量化效果

在 California Housing 数据集上的实验结果如下：

| 实验设置                      | RMSE     | 收敛 Epoch | 相对提升     |
|-----------------------------|----------|------------|--------------|
| Baseline（无变换）           | 68,000   | ~25        | —            |
| 归一化 + 标准化              | 58,000   | ~18        | ↑ 14.7%      |
| 加入对数变换（目标变量）     | 42,000   | ~15        | ↑ 38.2%      |

- 对数变换使 **RMSE 降低约 38%**  
- 归一化 + 标准化联合使用，**训练收敛速度提升约 40%**  
- 整体建模与实验效率提升 **3–5 倍**，支持高频策略迭代
